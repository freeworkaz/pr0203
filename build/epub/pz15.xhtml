Практическое занятие №15 Организация процесса построения защищенных каналов связи в различных сетях

Цель работы:

  1. Получить практический опыт применения инженерно-технических средств обеспечения информационной безопасности телекоммуникационных систем;
  2. Получить практический опыт применения программно-аппаратных средств обеспечения информационной безопасности телекоммуникационных систем;
  3. Получить практический опыт выявления технических каналов утечки информации.

Литература:

  1. Шаньгин, В. Ф. Информационная безопасность и защита информации [Электронный ресурс]: учебное пособие. - Москва: ДМК Пресс, 2014. -  Режим доступа: http://www.iprbookshop.ru/29257, по запросу, стр 463-478;

Подготовка к работе:

  1. Подготовить бланк отчета;
  2. Изучить предложенную литературу.

Основное оборудование:

  1. Компьютер под управлением ОС Windows Server 2008 R2;
  2. Виртуальная машина с дистрибутивом pfsense;
  3. Виртуальные машины в программе VirtualBox на основе Ubuntu Linux согласно заданию и схеме сети;
  4. Модель простой сети и ресурсов простой сети;
  5. Пакет GNS3 и комплект виртуальных машин, необходимых для схемы сети;
  6. Пакет программ LibreOffice.

Задание:

  1. Выполнить настройку защищенного соединения VPN при помощи ПАК pfsense в модели простой сети.

Порядок выполнения работы:

  1. Изучить схему простой сети. Выполнить подключение виртуальной машины pfsense в состав модели;
  2. Используя возможности програмно-аппаратного комплекса (ПАК) pfsense связать два компьютера модели с использованием VPN туннеля по протоколу PPTP;
  3. Проверить работу созданного туннеля с использованием программы wireshark. Для этого выполнить перехват пакетов на стороне клиента;
  4. Составить документ, в котором описать политику применения дистрибутива pfsense в рамках простейшей SOHO сети;
  5. Позвать преподавателя для проверки.

Содержание отчета:

  1. Цель работы;
  2. Содержание действий по выполнению данной работы;
  3. Ответы на контрольные вопросы;
  4. Вывод.

Контрольные вопросы:

  1. Каким образом осуществляется построение VPN туннелей с использованием выбранного ПАК?
  2. Каким образом осуществляется проверка работы созданного в выбранном ПАК туннеля?
  3. Каким образом происходит установление соединения в рамках туннеля, созданного с использованием выбранного ПАК?
