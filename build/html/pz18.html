Практическое занятие №18 Моделирование подсистем обеспечения безопасности информации от несанкционированного доступа¶

Цель работы:¶

  1. Получить практический опыт применения инженерно-технических средств обеспечения информационной безопасности телекоммуникационных систем;
  2. Получить практический опыт применения программно-аппаратных средств обеспечения информационной безопасности телекоммуникационных систем;
  3. Получить практический опыт выявления технических каналов утечки информации.

Литература:¶

  1. Шаньгин, В. Ф. Информационная безопасность и защита информации [Электронный ресурс]: учебное пособие. - Москва: ДМК Пресс, 2014. -  Режим доступа: http://www.iprbookshop.ru/29257, по запросу, стр 621-634;

Подготовка к работе:¶

  1. Подготовить бланк отчета;
  2. Изучить предложенную литературу.

Основное оборудование:¶

  1. Компьютер под управлением ОС Windows Server 2008 R2;
  2. Виртуальная машина с Ubuntu Linux;
  3. Веб сервер Apache Ubuntu Linux;
  4. Виртуальная машина с дистрибутивом pfsense;
  5. Модель сложной сети в программе GNS3;

Задание:¶

  1. Выполнить построение политики безопасности с учетом выполненных ранее заданий.

Порядок выполнения работы:¶

  1. Реализовать модель малой сети;
  2. Выполнить обследование ресурсов малой сети с выделением их роли в структуре сети. Выяснить реализованные механизмы безопасности. По необходимости просканировать ресурсы на предмет уязвимостей;
  3. Составить отчет об обследовании. По итогам отчета определить: #. Роли ресурсов в стурктуре предприятия. Результаты указать в отчете; #. Выделить активы для каждого их ресурса и определить их ценность. Результаты указать в отчете;
  4. Выделить реализованные механизмы безопасности, используемые в компонентах модели. Указать в отчете состав компонентов механизмов и реализованные настройки безопасности;
  5. Выполнить экспертную оценку эффективности системы защиты и обосновать ее преподавателю. Результаты указать в отчете;
  6. Выполнить анализ факторов, повышающих риски безопасности. Указать их в отчете;
  7. Выполнить сканирование ресурсов на уязвимости. Указать их в отчете;
  8. Выяснить влияние уязвимостей на систему безопасности и активы. Результаты указать в отчете;
  9. Составить классификацию уязвимостей по степени их опасности. Результаты указать в отчете;
  10. Составить матрицу угроз для каждого из активов. Результаты указать в отчете;
  11. Выполнить анализ рисков по одной из методик. Результаты указать в отчете;
  12. Составить модель нарушителя. Результаты указать в отчете;
  13. Оценить взаимосвязь всех элементов безопасности модели (активов, механизмов защиты, угроз, рисков, модели нарушителя). Результаты указать в отчете;
  14. На основе полученной в итоге информации составить политику безопасности. Результаты указать в отчете;
  15. Составить набор рекомендаций по обеспечению отказоустойчивости системы и политики резервного копирования. Результаты указать в отчете;
  16. Выработать рекомендации по улучшению системы защиты с учетом сотавленной политики безопасности. Результаты указать в отчете.
  17. Продемонстрировать отчет преподавателю.

Содержание отчета:¶

  1. Цель работы;
  2. Содержание действий по выполнению данной работы;
  3. Ответы на контрольные вопросы;
  4. Вывод.

  2. Что должна включать в себя политика безопасности предприятия?
  3. Что понимают под матрицей угроз безопасности?
