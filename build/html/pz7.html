Практическое занятие №7 Составление политики безопасности предприятия с учетом работы систем обнаружения вторжений.¶

Цель работы:¶

  1. Получить практический опыт выявления технических каналов утечки информации.

Литература:¶

  1. Хорев, П.Б. Программно-аппаратная защита информации : учебное пособие для вузов / П.Б. Хорев. - Москва: ФОРУМ, стр 69-96.

Подготовка к работе:¶

  1. Подготовить бланк отчета;
  2. Изучить предложенную литературу.

Основное оборудование:¶

  1. Компьютер под управлением ОС Windows Server 2008 R2;
  2. Виртуальные машины в программе VirtualBox на основе Ubuntu Linux согласно заданию и схеме сети;
  3. Модель простой сети и ресурсов простой сети;
  4. Пакет GNS3 и комплект виртуальных машин, необходимых для схемы сети;
  5. ОС Ubuntu Liunux с установленной СОВ Snort и настроенным интерфейсом;
  6. Пакеты прикладных программ LibreOffice.

Задание:¶

  1. Выполнить интеграцию СОВ Snort в модель сети;

Порядок выполнения работы:¶

  1. Выполнить подключение СОВ Snort в модель сети. Выполнить настройку интерфеса СОВ Snort.
  2. Составить политику безопасности с учетом использования СОВ Snort. В политике отразить:
  1. Активы и их роли в информационной систем предприятия.
  2. Уязвимости и угрзозы безопасности для выделенных ативов.
  3. Косвенные меры защиты, использующиеся для защиты активов. Их эффективность и состав.
  4. Прямые меры защиты, использующиеся для защиты активов. Их эффективность и состав.
  5. Сетевые угрозы для выделенных активов, их влияние на активы предприятия.
  6. Мера обнаружения сетевых угроз. Их эффективность. Порядок действий при обнаружении угрозы.
  7. Меры устранения угроз. Их эффективность. Порядок действий для устранения угрозы.
  8. Меры компенсирования воздействия угроз. Их эффективность. Порядок применения компенсационных мер для снижения результатов реализации угрозы.
  1. Построенную политику свести к таблице, на основе которой составить задание для администратора, в котором указать задачи администратора по выполнению вышеуказзанных пунктов политики.

Содержание отчета:¶

  1. Цель работы;
  2. Содержание действий по выполнению данной работы;
  3. Ответы на контрольные вопросы;
  4. Вывод.

Контрольные вопросы:¶

  1. Что понимают под системой защиты информации с «полным перекрытием»?
  2. Что понимают под мерами компенсирования воздействия угроз?
  3. Каким образом на практике оценивается эффективность реализованных мер защиты?

Приложение:¶

Установка и базовая настройка Snort, IDS устанавливается на тот же сервер, чей сетевой трафик и требуется проверять.

# sudo apt-get install snort-mysql

при установке будет запрос на ввод сети, которая будет считаться локальной, установить адрес локальной сети.

дальше последует пункт с созданием БД MySQL для хранения данных приложения, выполняем его как нам и подсказывает инсталлятор

# cd /usr/share/doc/snort-mysql/ # zcat create_mysql.gz | mysql -u <user> -h <host> -p <databasename>

Порядок настройки Snort описан в файле /usr/share/doc/snort-mysql/ в файде README.mysql. В данном файле расписаны основные команды по настройке СУБД Mysql для работы Snort создаем sql пользователя для доступа к этой базе и прописываем данные для доступа к этой БД в файл /etc/snort/database.conf командой sudo nano /etc/snort/database.conf

output database: log, mysql, user=<созданный пользователь> password=<ваш пароль> dbname=<созданная БД> host=localhost

удаляем файл /etc/snort/db-pending-config и запускаем Snort

sudo invoke-rc.d snort start

Устанавливаем Acidbase, веб-приложение для работы с данными собираемыми Snort’ом

sudo apt-get install acidbase

в процессе инсталляции будет надо будет указать пароль для доступа к БД Snort’а. Данные будут прописаны сюда /etc/acidbase/database.php

проходим по http://<IP_адрес>/acidbase/base_db_setup.php

жмем кнопку “Create BASE AG”, добавятся новые таблицы расширяющие БД Snort для поддержки функциональности BASE (так же будет напоминание, что для поддержки возможности удаления логов, sql пользователь из под которого ведется работа с БД должен иметь права delete и update)

проходим по http://<IP_адрес>/acidbase

Теперь можно создавать учетные записи в Acidbase, разграничивать права пользователей и продолжать работу с приложением.
